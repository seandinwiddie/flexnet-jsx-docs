HTTP System

This section outlines the FlexNet HTTP System, providing a functional API for making HTTP requests with zero external dependencies.

  • Overview: Functional implementation with immutable state, effect isolation, and zero dependencies.
  • Core Types: Request, Response, Cache, Interceptor types with strict functional definitions.
  • HTTP Client API: Client creation, making GET/POST/PUT/DELETE requests, using interceptors, and cache operations.
  • Type Verification: Strict verification of request, response, cache, and interceptor types.
  • Error Handling: All operations return an Either type for proper error management.
  • Effect Isolation: Side effects are fully isolated within the effect system.
