Getting Started with FlexNet JSX

Welcome to the getting started guide. This tutorial will walk you through setting up a project and building your first FlexNet JSX application.

1. Project Setup

The easiest way to start is by cloning one of our starter projects. For this guide, we'll use the minimal starter. 

# Clone the minimal starter project
git clone https://github.com/flexnet/starter-project-minimal.git flexnet-tutorial

# Change into the new directory
cd flexnet-tutorial

The project contains a basic file structure: 

.
├── index.html
└── src/
    ├── core/
    ├── systems/
    └── index.js

2. Running the Development Server

FlexNet JSX uses native JavaScript modules, so you need to serve your files from a local web server. A simple way to do this is with Python's built-in server. 

# If you have Python 3 installed
python -m http.server 3000

Now, you can open http://localhost:3000 in your browser. You should see a blank page.

3. Building Your First Component

Let's create our first component. Open the main application file at src/index.js and replace its contents with the following: 

import { createElement, render } from './systems/render/functions.js';

// A component is just a function that returns a UI description
const App = () => {
    return createElement(
        'div',
        { style: 'padding: 20px; font-family: sans-serif;' },
        createElement('h1', { style: 'color: #3b82f6;' }, 'Hello, FlexNet!'),
        createElement('p', null, 'Your first application is running.')
    );
};

// Find the root DOM element
const rootElement = document.getElementById('root');

// Render our App component into the root element
render(createElement(App), rootElement);

Make sure your index.html has a root element and includes the script: 

<!-- index.html -->
<body>
    <div id="root"></div>
    <script type="module" src="./src/index.js"></script>
</body>

Save your changes and refresh your browser. You should now see your "Hello, FlexNet!" message.
