HTTP System

FlexNet JSX does not include a built-in HTTP client. Instead, it relies on the browser's native fetch API and integrates with the state and effect systems to manage asynchronous network requests.

The Recommended Pattern

The standard way to handle HTTP requests is to use the Effect System. This pattern keeps your data flow predictable and separates the impure API call from your pure rendering logic.

  1. Dispatch a Request Action: A component dispatches an action to signal the start of a request (e.g., { type: 'FETCH_USER_REQUEST', payload: { id: 123 } }).
  2. Handle in an Effect: An effect listens for this action type, makes the fetch call, and handles the promise.
  3. Dispatch Result Actions: Based on the outcome of the fetch call, the effect dispatches a success (e.g., FETCH_USER_SUCCESS) or failure (FETCH_USER_FAILURE) action with the resulting data or error.
  4. Update State in Reducer: The reducer handles these result actions to update the state (e.g., setting a loading flag, storing the user data, or storing the error message).
// Example API effect
const apiEffect = (store, action) => {
    if (action.type === 'FETCH_USER_REQUEST') {
        store.dispatch({ type: 'SET_LOADING', payload: true });

        fetch(`/api/users/${action.payload.id}`)
            .then(res => {
                if (!res.ok) throw new Error('Network response was not ok');
                return res.json();
            })
            .then(data => {
                store.dispatch({ type: 'FETCH_USER_SUCCESS', payload: data });
                store.dispatch({ type: 'SET_LOADING', payload: false });
            })
            .catch(error => {
                store.dispatch({ type: 'FETCH_USER_FAILURE', payload: error.message });
                store.dispatch({ type: 'SET_LOADING', payload: false });
            });
    }
};
