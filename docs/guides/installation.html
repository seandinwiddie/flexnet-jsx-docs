Installation

FlexNet JSX is designed for simplicity. With zero external dependencies, you can get started by including a single script file in your project or by cloning one of our starter templates.

Option 1: Using the Starter Projects

This is the recommended approach for most users. We provide several starter projects to help you get up and running quickly.

Full-Featured Starter

A comprehensive implementation demonstrating all framework features, including state management, effects, and routing.

# Clone the repository
git clone https://github.com/flexnet/starter-project.git

# Navigate to the project directory
cd starter-project

# Serve the application
python -m http.server 3000

# Open http://localhost:3000 in your browser

Minimal Starter

A lightweight implementation focusing on core rendering concepts, perfect for learning the basics.

# Clone the repository
git clone https://github.com/flexnet/starter-project-minimal.git
cd starter-project-minimal
python -m http.server 3000

Option 2: Manual Setup

If you prefer to integrate FlexNet JSX into an existing project, you can do so by including the core script file.

  1. Download the latest release from the GitHub releases page.
  2. Extract the contents and locate the src directory.
  3. Copy the src directory into your project.
  4. Include the main JSX runtime file in your HTML:
<!-- This file includes all core systems needed to run FlexNet -->
<script type="module" src="path/to/your/src/index.js"></script>

Important Note

FlexNet JSX uses native ES modules, so you must serve your files from a web server (like python -m http.server or Live Server for VS Code). Opening the HTML file directly from your filesystem will not work.
