Installation

FlexNet JSX is designed for a straightforward setup with zero external dependencies. The recommended way to start is by cloning the official repository and using one of our pre-configured starter projects.

Core Principle: Zero Dependencies

FlexNet JSX uses only browser-native implementations and has no reliance on package managers like NPM or Yarn, a core principle defined in our Consistency Analysis. Installation is as simple as getting the code and serving it.

Using the Starter Projects (Recommended)

The starter projects provide a complete, correctly structured foundation for your application. They are the quickest and most reliable way to begin developing with FlexNet JSX, following the patterns in the Quick Start guide.

Step 1: Get the Repository

First, clone the official FlexNet JSX repository to get access to the starter projects and the full framework source.

git clone https://github.com/flexnet/flexnet-jsx.git
cd flexnet-jsx

Step 2: Choose a Starter and Serve

Two starter projects are available inside the cloned repository. Choose one and serve it locally.
