Quick Start

Build your first FlexNet JSX application. This guide will walk you through setting up a project and creating a simple interactive counter.

1. Project Setup

First, create a new directory for your project. Inside it, create the following folder and file structure. This organizes your code into features and keeps your static assets separate.

/my-app
├── /public
│   └── index.html
└── /src
    └── main.js

2. Create the HTML Entry Point

Your public/index.html file is the entry point for your application. It needs a root element for FlexNet JSX to mount into and a script tag to load your application code.

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>FlexNet JSX Counter</title>
</head>
<body>
    <div id="root"></div>
    <script type="module" src="../src/main.js"></script>
</body>
</html>

3. Build Your First Application

Now, let's build the interactive counter. This example demonstrates all the core concepts of FlexNet JSX working together: state management, functional components, and rendering. Place the following code in your src/main.js file.

// NOTE: Adjust these import paths based on where the framework is located.
import { createStore } from '../../systems/state/functions.js';
import { createElement, render } from '../../systems/render/functions.js';

// 1. STATE: Define a reducer for state transitions
const counterReducer = (state = 0, action) => {
  switch (action.type) {
    case 'INCREMENT':
      return state + 1;
    case 'DECREMENT':
      return state - 1;
    default:
      return state;
  }
};

// 2. STORE: Create a store to hold your application state
const store = createStore(counterReducer, 0);

// 3. VIEW: Create a component as a pure function
const Counter = ({ count, onIncrement, onDecrement }) => {
  return createElement('div', null,
    createElement('h1', null, `Count: ${count}`),
    createElement('button', { onClick: onIncrement }, '+'),
    createElement('button', { onClick: onDecrement }, '-')
  );
};

// 4. RENDER: Create a function that renders the UI based on the current state
const renderApp = () => {
  const state = store.getState();
  render(
    createElement(Counter, {
      count: state,
      onIncrement: () => store.dispatch({ type: 'INCREMENT' }),
      onDecrement: () => store.dispatch({ type: 'DECREMENT' })
    }),
    document.getElementById('root')
  );
};

// 5. SUBSCRIBE: Re-render the app whenever the state changes
store.subscribe(renderApp);

// Initial render
renderApp();

Step-by-Step Explanation

  1. State (Reducer)

    We define a `counterReducer`, a pure function that takes the current state and an action, and returns the next state. This is the only place where state logic lives.

  2. Store

    We create a `store` using `createStore`, passing it our reducer and the initial state (0). The store holds our state and allows us to dispatch actions.

  3. View (Component)

    Our `Counter` component is a pure function. It receives the current `count` and functions to call (`onIncrement`, `onDecrement`) as props, and returns a description of the UI.

  4. Render

    The `renderApp` function gets the latest state from the store and renders the `Counter` component with the appropriate props. The event handlers on the buttons will `dispatch` actions to the store when clicked.

  5. Subscribe

    We `subscribe` the `renderApp` function to the store. Now, every time an action is dispatched and the state changes, `renderApp` will be called automatically to update the UI.

Unidirectional Data Flow

This example demonstrates the core principle of unidirectional data flow in FlexNet JSX:
Action → Store → State → View
This makes the application's logic predictable and easy to follow.
