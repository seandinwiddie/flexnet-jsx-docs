Core Type System

FlexNet JSX implements a functional type system inspired by algebraic data types. These types help handle common programming challenges like optional values and error handling in a pure, functional way, as described in the Architecture Overview.

Looking for In-Depth Usage?

This page provides a high-level summary of the core types. For detailed explanations, usage patterns, and best practices, please see the complete Error Handling Guide.

The Core Types

The FlexNet JSX type system is built on three fundamental container types. They follow the principles of functional programming, particularly the concept of monads, which allows for composing operations in a safe and predictable manner.

Maybe

For handling optional values that could be `null` or `undefined`, eliminating a common source of runtime errors.

→ View API Reference → See Usage in Guide

Either

For representing a value that can be one of two distinct possibilities, typically a success (`Right`) or a failure (`Left`). It is ideal for explicit error handling.

→ View API Reference → See Usage in Guide

Result

Similar to `Either`, but specifically designed for operations that can succeed (`Ok`) or fail with a native `Error` object (`Error`), making it perfect for wrapping exception-throwing code.

→ View API Reference → See Usage in Guide
