Type System

FlexNet JSX implements a functional type system inspired by algebraic data types from functional programming languages. These types help handle common programming challenges like optional values, error handling, and operation outcomes in a pure functional way.

Note

FlexNet's type system is designed to be used with JavaScript's native capabilities, without requiring a compile-time type checker like TypeScript or Flow. However, it's fully compatible with these tools if you choose to use them.

Core Types Overview

FlexNet JSX includes three fundamental types that form the backbone of the type system:

  • Maybe: For handling optional values and eliminating null checks.
  • Either: For handling branching logic and error cases with explicit paths.
  • Result: For capturing operation outcomes and handling exceptions functionally.

These types follow the principles of functional programming, particularly the concept of monads, which allows for composition and chaining of operations in a safe and predictable manner.

Maybe Type

The Maybe type is a container for a value that may or may not be present. It helps you avoid `null` or `undefined` errors by forcing you to explicitly handle the case where a value is missing.

Structure

// src/core/types/maybe.js
const Maybe = {
    Just: value => ({ type: 'Just', value }),
    Nothing: () => ({ type: 'Nothing' }),
    fromNullable: value => value != null ? Maybe.Just(value) : Maybe.Nothing(),
    map: fn => maybe => 
        maybe.type === 'Just' ? Maybe.Just(fn(maybe.value)) : Maybe.Nothing(),
    chain: fn => maybe =>
        maybe.type === 'Just' ? fn(maybe.value) : Maybe.Nothing(),
    getOrElse: defaultValue => maybe =>
        maybe.type === 'Just' ? maybe.value : defaultValue
};

Key Concepts

  • Just(value) - A container that holds a value.
  • Nothing() - A container that represents the absence of a value.
  • fromNullable(value) - Creates a `Maybe` from a value that could be `null` or `undefined`.
  • map(fn) - Applies a function to the value inside a `Just`, otherwise returns `Nothing`.
  • chain(fn) - Applies a function that returns a `Maybe` to the value inside a `Just`. Useful for chaining operations that can fail.
  • getOrElse(defaultValue) - Extracts the value from a `Just` or returns a default value if it's `Nothing`.

Usage Example

import { Maybe } from '../../core/types/maybe.js';

// Safely access a nested property
const getStreetName = user =>
  Maybe.fromNullable(user)
    .map(u => u.address)
    .map(a => a.street)
    .getOrElse('No street found');

// Usage
const user1 = { address: { street: '123 Main St' } };
const user2 = { address: null };
const user3 = null;

console.log(getStreetName(user1)); // '123 Main St'
console.log(getStreetName(user2)); // 'No street found'
console.log(getStreetName(user3)); // 'No street found'

For complete details on the Maybe type, see the Maybe API Reference.

Either Type

The Either type represents a value that can be one of two possibilities, conventionally named `Left` and `Right`. It is typically used for error handling, where `Left` represents an error and `Right` represents a successful value.

Structure

// src/core/types/either.js
const Either = {
    Left: value => ({ type: 'Left', value }),
    Right: value => ({ type: 'Right', value }),
    fromNullable: value => value != null ? Either.Right(value) : Either.Left('Value is null'),
    map: fn => either =>
        either.type === 'Right' ? Either.Right(fn(either.value)) : either,
    chain: fn => either =>
        either.type === 'Right' ? fn(either.value) : either,
    fold: (leftFn, rightFn) => either =>
        either.type === 'Right' ? rightFn(either.value) : leftFn(either.value)
};

Key Concepts

  • Left(value) - Represents the "failure" or alternative path, containing an error value.
  • Right(value) - Represents the "success" path, containing the correct value.
  • map(fn) - Applies a function to the value inside a `Right`, leaving a `Left` untouched.
  • chain(fn) - Applies a function that returns an `Either` to the value of a `Right`.
  • fold(leftFn, rightFn) - Provides two functions to handle both cases. It safely unwraps the `Either` by executing the appropriate function.

Usage Example

import { Either } from '../../core/types/either.js';

// A function that can fail
const findUser = id => id === 1
  ? Either.Right({ id: 1, name: 'Alice' })
  : Either.Left('User not found');

// Process the result using fold
const getUserInfo = id =>
  findUser(id)
    .map(user => user.name.toUpperCase())
    .fold(
      error => `Error: ${error}`,
      name => `User: ${name}`
    );

// Usage
console.log(getUserInfo(1)); // 'User: ALICE'
console.log(getUserInfo(2)); // 'Error: User not found'

For complete details on the Either type, see the Either API Reference.

Result Type

The Result type is similar to `Either` but is specifically designed for operations that can succeed (`Ok`) or fail (`Error`). It is excellent for wrapping native JavaScript functions that might throw exceptions, bringing them into the functional world.

Structure

// src/core/types/result.js
const Result = {
    Ok: value => ({ type: 'Ok', value }),
    Error: error => ({ type: 'Error', error }),
    fromTry: fn => {
        try {
            return Result.Ok(fn());
        } catch (e) {
            return Result.Error(e);
        }
    },
    map: fn => result =>
        result.type === 'Ok' ? Result.Ok(fn(result.value)) : result,
    chain: fn => result =>
        result.type === 'Ok' ? fn(result.value) : result
};

Key Concepts

  • Ok(value) - Represents a successful operation, containing the resulting value.
  • Error(error) - Represents a failed operation, containing the error or exception.
  • fromTry(fn) - Executes a function and wraps its outcome in a `Result`, catching any exceptions and placing them in `Error`.
  • map(fn) - Applies a function to the value inside an `Ok` container.

Usage Example

import { Result } from '../../core/types/result.js';

// Safely parse a JSON string that might be invalid
const parseJson = jsonString => Result.fromTry(() => JSON.parse(jsonString));

// Usage
const validJson = '{"data": "value"}';
const invalidJson = 'not json';

const result1 = parseJson(validJson).map(data => data.data); // Result.Ok("value")
const result2 = parseJson(invalidJson); // Result.Error(SyntaxError: ...)

result1.map(value => console.log(value)); // Logs "value"
result2.map(value => console.log(value)); // Does nothing
