JSX Runtime

The FlexNet JSX Runtime provides a functional implementation for declaring user interfaces. It transpiles JSX syntax into pure function calls that build a virtual DOM tree, which is then efficiently rendered to the browser.

A Pure Functional Approach

Unlike other frameworks, FlexNet JSX is built entirely on pure functions, immutability, and composition. It has zero external dependencies and integrates seamlessly with the framework's core Type System.

Core API

The JSX runtime is powered by two main functions located in src/systems/render/functions.js.

createElement

The createElement function is what your JSX code gets converted into. It takes a type, props, and children, and returns a lightweight JavaScript object known as a virtual DOM node. 

const createElement = (type, props, ...children) => ({
    type,
    props: { ...props, children: children.flat() }
});

// JSX like:
// <div className="container">Hello World</div>

// Is transpiled to:
createElement('div', { className: 'container' }, 'Hello World');

render

The render function is a pipeline that takes a virtual DOM element and makes it appear in the browser. It creates a complete virtual DOM tree, reconciles it with the existing DOM, and applies only the necessary changes (patches). 

import { pipe } from '../../core/functions/composition.js';
// NOTE: createVirtualDOM, reconcile, and patch are internal functions.

const render = (element, container) =>
    pipe(
        createVirtualDOM,
        reconcile(container.firstChild),
        patch(container)
    )(element);

Functional Components

In FlexNet, components are simply pure functions that accept props and return a virtual DOM element. This makes them highly predictable, testable, and composable. 

import { createElement } from '../../systems/render/functions.js';

// A pure functional component
const Greeting = ({ name }) =>
  createElement('h1', null, `Hello, ${name}!`);

// Using the component produces a VDOM node
const element = createElement(Greeting, { name: 'World' });

// This is the equivalent of writing:
// <Greeting name="World" />

Rendering Example

import { createElement, render } from '../../systems/render/functions.js';

// A pure component function
const App = ({ message }) =>
  createElement('div', { className: 'app' },
    createElement('h1', null, 'FlexNet JSX'),
    createElement('p', null, message)
  );

// The render function mounts the component to the DOM
render(
  createElement(App, { message: 'Declarative UI with pure functions!' }),
  document.getElementById('root')
);

Integration with the Type System

The JSX runtime is designed to work seamlessly with FlexNet's functional types to handle conditional rendering and optional props safely. 

import { createElement, render } from '../../systems/render/functions.js';
import { Maybe } from '../../core/types/maybe.js';

// Using Maybe for an optional prop
const UserProfile = ({ user }) => {
  const username = Maybe.fromNullable(user)
    .map(u => u.name)
    .getOrElse('Guest');

  return createElement('div', { className: 'profile' },
    createElement('h2', null, `Welcome, ${username}!`)
  );
};

// Render the component with different props
const loggedInUser = { name: 'Alice' };
render(createElement(UserProfile, { user: loggedInUser }), document.getElementById('user1'));
render(createElement(UserProfile, { user: null }), document.getElementById('user2'));

Key Aspects of FlexNet JSX

  • Purely Functional: Components are pure functions, ensuring predictable rendering and easy testing. Class components are not used.
  • Lightweight VDOM & Reconciliation: FlexNet uses a virtual DOM and a reconciliation pipeline to efficiently update the UI. The entire process is a pure functional composition.
  • Integrated Type System: The runtime is built to leverage the core functional types (`Maybe`, `Either`, `Result`) for robust and safe UI logic.
  • Zero Dependencies: The framework has no external dependencies, resulting in a small footprint and full control over the implementation.
