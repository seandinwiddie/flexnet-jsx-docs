Core Functions

FlexNet JSX is built on functional programming principles. It provides a set of core utilities to compose functions and transform data in a pure, predictable way. These functions are designed to work seamlessly with the framework's Type System (Maybe, Either, and Result) to build robust applications.

Function Composition

FlexNet provides utilities for function composition, a central pattern in functional programming for combining smaller functions into more complex ones.

compose

The compose function allows you to combine multiple functions, applying them from right to left. It's a powerful way to chain operations where the output of one function becomes the input of the next.

import { compose } from '../../core/functions/composition.js';

// Example functions
const double = x => x * 2;
const increment = x => x + 1;
const square = x => x * x;

// Compose functions: square(double(increment(x)))
const process = compose(square, double, increment);

// Usage
process(3); // Result: 64
// Calculation: square(double(increment(3))) = square(double(4)) = square(8) = 64

pipe

The pipe function is similar to compose, but it applies functions from left to right. This can sometimes feel more natural as it follows the reading order of the code.

import { pipe } from '../../core/functions/composition.js';

// Example functions
const double = x => x * 2;
const increment = x => x + 1;
const square = x => x * x;

// Pipe functions: square(increment(double(x)))
const process = pipe(double, increment, square);

// Usage
process(3); // Result: 49
// Calculation: square(increment(double(3))) = square(increment(6)) = square(7) = 49

curry

The curry function transforms a function that takes multiple arguments into a sequence of functions that each take a single argument. This is useful for creating specialized functions through partial application.

import { curry } from '../../core/functions/composition.js';

// Original function with multiple arguments
const add = (a, b, c) => a + b + c;

// Curried version
const curriedAdd = curry(add);

// Usage
const result = curriedAdd(1)(2)(3); // Result: 6

// Partial application to create a specialized function
const add5 = curriedAdd(5);
const result2 = add5(2)(3); // Result: 10

Data Transformation

A core part of functional programming is transforming data structures. FlexNet provides higher-order functions for common array operations, which promote immutability by returning new arrays instead of modifying the original ones.

map

The map function takes a function and an array, and applies that function to every element in the array, returning a new array with the results.

import { map } from '../../core/functions/transforms.js';

const double = x => x * 2;
const numbers = [1, 2, 3];

const doubledNumbers = map(double)(numbers); // Result: [2, 4, 6]

filter

The filter function takes a predicate function and an array. It returns a new array containing only the elements for which the predicate returns `true`.

import { filter } from '../../core/functions/transforms.js';

const isEven = x => x % 2 === 0;
const numbers = [1, 2, 3, 4, 5, 6];

const evenNumbers = filter(isEven)(numbers); // Result: [2, 4, 6]

reduce

The reduce function (also known as fold) executes a reducer function on each element of the array, resulting in a single output value.

import { reduce } from '../../core/functions/transforms.js';

const sum = (acc, x) => acc + x;
const numbers = [1, 2, 3, 4, 5];

const total = reduce(sum, 0)(numbers); // Result: 15

Combining Functions and Types

The true power of FlexNet JSX comes from combining function composition with the core type system. This allows for creating safe, explicit, and resilient data processing pipelines.

import { pipe } from '../../core/functions/composition.js';
import { map } from '../../core/functions/transforms.js';
import { Either } from '../../core/types/either.js';

// A function that might fail
const parseNumber = str => {
    const num = parseFloat(str);
    return isNaN(num) ? Either.Left('Invalid number') : Either.Right(num);
}

// A pure function for an operation
const increment = n => n + 1;

// Use pipe to create a safe pipeline
const safeIncrement = pipe(
    parseNumber,
    Either.map(increment)
);

const result1 = safeIncrement('5');   // Either.Right(6)
const result2 = safeIncrement('foo'); // Either.Left('Invalid number')

result1.fold(
  error => console.error(error),
  value => console.log('Success:', value) // Logs 'Success: 6'
);

Utility Functions

FlexNet also provides helpful utilities that follow functional principles.

memoize

The memoize function creates a cached version of a function. It stores the results of previous calls and returns the cached result for the same input, avoiding expensive recalculations.

import { memoize } from '../../utils/function.js';

// An expensive computation
const slowFunction = (num) => {
  // Simulate a delay
  for (let i = 0; i < 1e9; i++) {}
  return num * 2;
};

const fastFunction = memoize(slowFunction);

console.time('first call');
fastFunction(5); // Takes a long time
console.timeEnd('first call');

console.time('second call');
fastFunction(5); // Returns instantly from cache
console.timeEnd('second call');

Pure Functions are Key

memoize works best with pure functions. Since pure functions always return the same output for the same input, their results can be safely cached without causing unexpected behavior.

Best Practice

When working with FlexNet JSX, prefer using pure functions and functional composition over imperative approaches. Combine `pipe` and `compose` with the `Maybe`, `Either`, and `Result` types to create data pipelines that are safe, readable, and easy to debug.
