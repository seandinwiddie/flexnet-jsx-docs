Maybe API

The Maybe type provides a functional and safe way to handle optional values, eliminating null and undefined errors.

Overview

The Maybe type is a container that represents a value that may or may not exist. It has two possible variants:

  • Just(value): An object representing the presence of a value, e.g., { type: 'Just', value: 42 }.
  • Nothing(): An object representing the absence of a value, e.g., { type: 'Nothing' }.

By using Maybe, you make the possibility of a missing value explicit in your function signatures. This forces you to handle the "nothing" case, preventing entire classes of runtime errors and making your code more robust and predictable.

Key Principles

  • Eliminate Null Errors: Replaces the need for defensive null/undefined checks throughout your code.
  • Explicit Handling: Forces you to consider both the "Just" and "Nothing" cases when using a value.
  • Composable Pipelines: Allows for safe, sequential operations on values that might be missing at any step.

Creating and Using a Maybe

Constructors

You can create a Maybe directly or from a potentially nullish value.

import { Maybe } from './src/core/types/maybe.js';

// Create a Maybe containing a value
const aJust = Maybe.Just(42); // { type: 'Just', value: 42 }

// Create an empty Maybe
const aNothing = Maybe.Nothing(); // { type: 'Nothing' }

// Safely create a Maybe from a variable that might be null or undefined
const user = { name: 'Alice' };
const noUser = null;

const maybeUser = Maybe.fromNullable(user); // Just({ name: 'Alice' })
const maybeNoUser = Maybe.fromNullable(noUser); // Nothing()

Safely Accessing the Value

The safest way to get a value out of a Maybe is with getOrElse, which provides a default value if the Maybe is Nothing.

import { Maybe } from './src/core/types/maybe.js';

const maybeName = Maybe.fromNullable(user.name);
const maybeMissingName = Maybe.Nothing();

const getName = Maybe.getOrElse('Guest');

console.log(getName(maybeName)); // 'Alice'
console.log(getName(maybeMissingName)); // 'Guest'

Composing Operations

The true power of Maybe comes from its composable functions, which allow you to build safe data pipelines.

Transforming a Value with `map`

The map function applies a transformation to the value inside a Just and does nothing to a Nothing.

import { Maybe } from './src/core/types/maybe.js';
import { pipe } from './src/core/functions/composition.js';

const config = { port: 8080 };
const noConfig = null;

const getPort = (c) => c.port;
const increment = (n) => n + 1;

const getNextPort = pipe(
    Maybe.map(getPort),    // Maybe({ port: 8080 }) -> Maybe(8080)
    Maybe.map(increment) // Maybe(8080) -> Maybe(8081)
);

const nextPort = getNextPort(Maybe.fromNullable(config)); // Just(8081)
const noNextPort = getNextPort(Maybe.fromNullable(noConfig)); // Nothing()

console.log(Maybe.getOrElse(3000)(nextPort)); // 8081
console.log(Maybe.getOrElse(3000)(noNextPort)); // 3000

Sequencing with `chain`

Use chain to sequence operations where each step returns a new Maybe. This is perfect for safely accessing nested properties.

import { Maybe } from './src/core/types/maybe.js';
import { pipe } from './src/core/functions/composition.js';

const user = {
  name: 'John',
  address: {
    street: '123 Main St'
  }
};

const getAddress = (u) => Maybe.fromNullable(u.address);
const getStreet = (a) => Maybe.fromNullable(a.street);

const getStreetName = pipe(
    Maybe.chain(getAddress), // Maybe(user) -> Maybe(address)
    Maybe.chain(getStreet)   // Maybe(address) -> Maybe(street)
);

const street = getStreetName(Maybe.fromNullable(user)); // Just('123 Main St')

console.log(Maybe.getOrElse('Street not found')(street)); // '123 Main St'

// Example with missing data
const userWithoutAddress = { name: 'Jane' };
const missingStreet = getStreetName(Maybe.fromNullable(userWithoutAddress)); // Nothing()

console.log(Maybe.getOrElse('Street not found')(missingStreet)); // 'Street not found'

API Reference

Constructors

Constructor Description
Maybe.Just(value) Creates a Maybe container for an existing value.
Maybe.Nothing() Creates an empty Maybe container representing a missing value.
Maybe.fromNullable(value) Creates a Just if the value is not null or undefined, otherwise returns Nothing.

Functions

Function Description
map(fn) A curried function that applies fn to the value inside a Just, otherwise returns Nothing.
Signature: (fn: A -> B) => (maybe: Maybe) => Maybe
chain(fn) A curried function that applies fn to the value inside a Just, where fn must return a new Maybe.
Signature: (fn: A -> Maybe) => (maybe: Maybe) => Maybe
getOrElse(defaultValue) A curried function that unwraps a Just to get its value, or returns defaultValue if it's a Nothing.
Signature: (defaultValue: A) => (maybe: Maybe) => A

Framework Compliance

  • Use Curried Functions: The functions map, chain, and getOrElse are not instance methods. They are curried functions designed to be used with composition tools like pipe.
  • Check the Type Property: For simple conditional logic, you can directly check maybe.type which will be either 'Just' or 'Nothing'.
  • Synchronous Only: Maybe is for handling optional values in synchronous code. It is not designed for asynchronous operations.
