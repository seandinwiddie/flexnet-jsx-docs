Store API

The Store API provides a pure functional, immutable state management solution, fully compliant with the FlexNet JSX Framework's core principles.

Overview

The Store is a container for your application's state, built on a foundation of pure functional programming. It guarantees that state is always predictable and traceable.

  • Immutable State: State is never changed directly. All updates produce a new state.
  • Pure Reducers: State transitions are managed by pure functions called reducers.
  • Predictable Flow: State changes are centralized and occur in a strict, one-way data flow.
  • Type-Safe by Design: Leverages the framework's core types like Maybe, Either, and Result for robust state modeling.

Core Concepts

  • State: An immutable data structure representing the condition of your app.
  • Action: A plain object describing a requested change to the state.
  • Reducer: A pure function that takes the current state and an action, then returns the next state.
  • Dispatch: The function used to send an action to the reducer.
  • Subscriber: A function that is called whenever the state changes.

Creating a Store

A Store is created with the createStore function, which requires a reducer and an initial state.

import { createStore } from './src/systems/state/functions.js';
import { pipe } from './src/core/functions/composition.js';

// 1. Define actions
const Action = {
  Increment: 'INCREMENT',
  Decrement: 'DECREMENT'
};

// 2. Define a reducer: state => action => newState
const counterReducer = state => action => {
    switch (action.type) {
        case Action.Increment:
            return state + 1;
        case Action.Decrement:
            return state - 1;
        default:
            return state;
    }
};

// 3. Create a store with the reducer and initial state
const counterStore = createStore(counterReducer, 0);

Reading State

The current state can be retrieved at any time using getState().

// Get the current state
const count = counterStore.getState();
console.log(count); // 0

Dispatching Actions

To change the state, you must dispatch an action. This is the only way to trigger a state update.

// Dispatch an action to increment the counter
counterStore.dispatch({ type: Action.Increment });
console.log(counterStore.getState()); // 1

// Dispatch another action
counterStore.dispatch({ type: Action.Increment });
console.log(counterStore.getState()); // 2

// Dispatch an action to decrement
counterStore.dispatch({ type: Action.Decrement });
console.log(counterStore.getState()); // 1

Subscribing to Changes

You can listen for state changes using subscribe(). It takes a callback that executes whenever the state is updated and returns a function to unsubscribe.

// Subscribe to state changes
const unsubscribe = counterStore.subscribe(() => {
  const newState = counterStore.getState();
  console.log('State changed:', newState);
  // This is where you would trigger a re-render of your UI
});

// Dispatching an action will now trigger the subscriber
counterStore.dispatch({ type: Action.Increment }); // Logs: "State changed: 2"

// To stop listening, call the returned function
unsubscribe();

counterStore.dispatch({ type: Action.Decrement }); // No log, because we unsubscribed

Advanced Usage: Type-Safe State

A key strength of FlexNet JSX is integrating the core type system into state management. This example uses Maybe to model an optional user session.

import { createStore } from './src/systems/state/functions.js';
import { Maybe } from './src/core/types/maybe.js';

const UserAction = {
  Login: 'LOGIN',
  Logout: 'LOGOUT',
  UpdateAge: 'UPDATE_AGE'
};

const userReducer = maybeUser => action => {
    switch (action.type) {
        case UserAction.Login:
            // The payload is the user object
            return Maybe.Just(action.payload);
        case UserAction.Logout:
            return Maybe.Nothing();
        case UserAction.UpdateAge:
            // Use .map to safely update the user if it exists
            return maybeUser.map(user => ({
                ...user,
                age: user.age + 1
            }));
        default:
            return maybeUser;
    }
};

// The initial state is Nothing, as no user is logged in
const userStore = createStore(userReducer, Maybe.Nothing());

// Dispatch login action
userStore.dispatch({
    type: UserAction.Login,
    payload: { name: 'Jane Doe', age: 30 }
});

// getState() now returns a Maybe.Just
userStore.getState().map(user => console.log(user.name)); // "Jane Doe"

// Dispatch logout action
userStore.dispatch({ type: UserAction.Logout });

// getState() now returns Maybe.Nothing
console.log(userStore.getState().isNothing()); // true

Store API Reference

createStore

Signature

function createStore(reducer: (state: S) => (action: A) => S, initialState: S): Store;

Parameters

Name Type Description
reducer (state: S) => (action: A) => S A pure, curried function that returns the next state, given the current state and an action.
initialState S The initial state of the store.

Returns

A new Store instance with three methods: getState, dispatch, and subscribe.

Store Methods

getState()

Returns the current state of the store.

function getState(): S;

dispatch(action)

Dispatches an action to the reducer, triggering a state update.

function dispatch(action: A): void;

subscribe(subscriber)

Adds a listener that will be called after every dispatch.

function subscribe(subscriber: () => void): () => void;

Returns an unsubscribe function that can be called to remove the subscription.

Framework Compliance

  • Pure Functions Only: Reducers must be pure. Given the same state and action, they must always return the same new state.
  • Immutability: Never mutate the state object or its fields. Always return a new object.
  • No Side Effects: Reducers must not have side effects (e.g., API calls, timeouts). Side effects belong in the Effect System.
  • Functional Composition: Use compose and pipe to build more complex reducers or state transformations.
